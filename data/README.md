# Data Directory

This directory contains SQLite database files and other data generated by the player tracker.

## Files

- `player_tracker.db` - Main database storing player sessions and statistics
- `player_tracker_*.db` - Backup database files (timestamped)

## Database Schema

The database contains the following tables:

### train_sessions
Train driving sessions with distance, points, and detailed vehicle composition:
- **Basic info**: steam_id, server, train number/name, route, timestamps
- **Performance**: distance_meters, points, baseline stats
- **Vehicle composition** (new):
  - `vehicle_summary` - Human-readable summary (e.g., "EU07-002 + 15 wagons")
  - `traction_type` - LOCOMOTIVE, EMU, or MULTIPLE_UNIT
  - `locomotive_names` - Comma-separated list (e.g., "EU07-002, EU07-015" for double-headed)
  - `num_locomotives` - Number of locomotives
  - `num_wagons` - Number of wagons
  - `total_vehicles` - Total consist length
  - `total_length` - Total train length in meters
  - `total_weight` - Total train weight in tons
  - `composition_json` - Full composition details as JSON

### station_sessions
Dispatcher sessions at stations with timestamps

### train_station_passages
Stations passed during train sessions with stop types

### steam_stats
Historical Steam statistics snapshots

## Usage

View tracked data:
```bash
uv run scripts/show_summary.py <steam_id>
```

Query vehicle composition:
```sql
-- Find double-headed trains
SELECT train_number, locomotive_names, total_weight 
FROM train_sessions 
WHERE num_locomotives > 1;

-- Find heaviest trains
SELECT train_number, vehicle_summary, total_weight
FROM train_sessions
ORDER BY total_weight DESC
LIMIT 10;
```
